# 右侧悬浮 Git 与进程信息面板设计

## 目标

将现有右上角 Git 小面板升级为参考图风格的悬浮信息卡，同时展示 Git 状态和后台命令状态。
面板默认折叠，避免长期遮挡聊天内容；用户点击后展开完整信息。

## 范围

本次包含：

- 默认折叠的右上角状态标签。
- 参考图风格的展开卡片。
- Git 变更、分支和提交入口。
- 最新启动后台命令的状态。
- 最新命令的运行时长和当前运行中后台命令总数。
- 深色、浅色主题适配。
- 修复本次涉及组件中已经存在的乱码文本。

本次不包含：

- 后台进程历史列表或进程管理操作。
- 同时展开多条后台命令详情。
- 新增后台进程 IPC 协议。
- 实现完整的 Git 提交编辑器、推送流程或分支切换器。

## 交互设计

### 折叠态

面板默认折叠为右上角小标签，展示当前分支、是否存在变更以及展开箭头。标签应保持轻量，
不明显遮挡聊天标题和正文。

点击标签展开卡片。再次点击标签或点击卡片外部时收起。

### 展开态

展开卡片采用用户选择的“A · 参考图复刻”方向：

1. 顶部为“Git 工具”标题和辅助操作图标。
2. Git 区展示新增行数、删除行数、当前分支和提交入口。
3. 分隔线下方为进程区，展示最新启动的一条后台命令。
4. 底部展示该命令的运行时长和当前运行中的后台命令数量。

卡片贴近右上角悬浮，使用圆角、轻边框和阴影。颜色来自现有主题变量，并分别使用绿色和
红色表达新增与删除。

## 数据与状态

### Git 数据

继续调用现有 `bridge.getGitInfo(workingDirectory)`。活动会话或工作目录变化时刷新。

展示字段：

- `branch`：当前分支。
- `changedLines`：当前可用的变更行统计。
- `changedFiles`：变更文件数。
- `stagedFiles`：已暂存文件数。

若当前目录不是 Git 仓库，则不显示悬浮面板。Git 查询失败时使用安全的空状态，不影响聊天
界面。

现有 Git 接口若不能分别提供新增和删除行数，实施时应扩展 Git 信息结果为明确的
`addedLines` 和 `deletedLines`，由主进程完成统计并通过已类型化的 preload/IPC 边界返回。
不使用单一变更行数伪造正负数据。

### 后台命令数据

复用 `useBackgroundProcesses()` 中的共享进程集合和已有事件：

- `agent:bg-process-started`
- `agent:bg-process-completed`

“最新命令”按 `startTime` 最大值选择，不因较早命令晚结束而改变。

进程区显示：

- 命令文本。
- `running`、`completed` 或 `error` 状态。
- 运行中后台命令数。
- 最新命令的开始时间。

长命令单行截断，并通过 `title` 提供完整文本。

### 运行时长

运行时长从最新后台命令的 `startTime` 开始计算。运行期间每秒更新显示。

格式规则：

- 小于 1 分钟：`N秒`
- 小于 1 小时：`N分 M秒`
- 1 小时及以上：`N小时 M分`

命令结束后停止增长，并显示完成或失败状态。由于现有完成事件只包含 PID 和退出码，如需精确
冻结耗时，前端在收到完成事件时记录 `endTime`；该值仅用于当前渲染会话，无需持久化。

没有后台命令时显示“暂无运行进程”，运行时长位置显示“未运行 · 0 后台”。

## 组件边界

主要改动集中在 `src/renderer/components/layout/GitPanel.vue`：

- 管理折叠与展开状态。
- 获取和刷新 Git 信息。
- 读取共享后台进程状态。
- 计算最新命令、运行数量和格式化时长。
- 渲染折叠标签与展开卡片。

`useBackgroundProcesses.ts` 继续作为后台进程事件的共享数据源。若组件卸载逻辑会导致多个消费
者互相停止监听，应将监听生命周期调整为模块级单例引用计数或应用级常驻订阅，确保
`ChatHeader` 与 `GitPanel` 可同时消费同一数据。

若需要新增 Git 增删行字段，则同步更新：

- `src/shared/types.ts`
- `src/main/preload.ts`
- `src/main/ipc-handlers.ts`
- `src/renderer/api/bridge.ts`
- `src/renderer/types/ipc.ts`

所有跨进程数据保持显式类型。

## 可访问性与响应式

- 折叠按钮使用原生 `button`，提供明确的 `aria-expanded` 和中文 `aria-label`。
- 图标按钮提供 `title` 或 `aria-label`。
- 键盘可聚焦并通过 Enter、Space 展开或收起。
- 面板宽度使用固定理想值和视口上限，窄窗口中不超出主内容区域。
- 状态不只依赖颜色，同时显示文字或图标。

## 错误与空状态

- Git 查询失败：保留聊天功能，面板使用空状态或隐藏。
- 非 Git 仓库：隐藏整个 Git 悬浮面板。
- 无后台命令：显示明确空状态。
- 命令失败：显示失败状态和退出码（若存在）。
- 重复 PID 事件：按 PID 更新现有记录，不重复插入。

## 验证

自动验证：

- `bun run typecheck`
- `bun run lint`
- `bun run build`

手动验证：

- Git 仓库与非 Git 目录下的显示行为。
- 默认折叠、点击展开、再次点击和点击外部收起。
- 新增/删除行数、分支和变更数量显示正确。
- 启动一个后台命令后，最新命令、计时和后台数量实时更新。
- 多个后台命令同时运行时，只展示最新命令并正确汇总数量。
- 最新命令成功和失败结束后的状态与冻结时长。
- 深色和浅色主题。
- 窄窗口下卡片不越界。
- `ChatHeader` 与悬浮面板同时显示后台状态时，事件监听不丢失。
