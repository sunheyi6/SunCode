# 会话切换性能优化设计

## 目标

切换会话时优先展示最近 10 条消息；完整历史在后台异步加载并缓存，减少首次 IPC 返回数据和渲染阻塞，同时保证 Worker 能继续使用完整上下文。

## 方案

渲染层的 `sessions` store 使用每会话缓存。命中缓存时立即渲染 10 条；未命中时先请求 `maxMessages=10`，拿到快照后立即切换，并启动一次后台完整加载。完整加载结果仅在用户仍停留在目标会话时更新界面，并写入缓存，避免快速切换造成旧请求覆盖当前会话。

主进程区分“展示数据”和“Worker 数据”：`session:load` 返回给 Renderer 的消息严格遵守 `maxMessages`，但内部保存完整消息并发送给 Worker。已有内存消息不能绕过调用方的消息上限。

预热任务限制并发量，避免启动时同时读取所有会话文件。切换中的请求使用递增请求序号做竞态保护。

## 验证

- session store 单元测试验证缓存快照优先、完整加载异步补全、过期请求不会更新当前会话。
- session store 文件测试验证 `maxMessages` 只影响返回快照而不破坏完整内存数据。
- 运行相关 Vitest、Biome lint 和 TypeScript typecheck。
